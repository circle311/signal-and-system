## 4. Work Plan

### Phase 1: Setup and Input Selection

1. Use MATLAB as the main implementation language.
2. Select a short audio clip, preferably mono or converted to mono for simpler analysis.
3. Store the original audio under `audio/input/`.
4. Record sampling rate, duration, channel count, and any preprocessing choices in the report notes.

### Phase 2: Continuous-Time Counterpart Approximation

1. Denote the sampled audio as `x[n]`.
2. Use an interpolation method such as zero-order hold or first-order hold to construct an approximation of `x(t)`.
3. Plot and save the waveform of `x(t)`.
4. Explain the interpolation method and why it is suitable for this project.

### Phase 3: Time-Scaling and Time-Reversal Signals

1. Generate `x(-t)`, `x(2t)`, and `x(t/2)`.
2. Save the corresponding audio clips when meaningful.
3. Plot and save their time-domain waveforms.
4. Note expected perceptual effects:
   - `x(-t)`: reversed audio.
   - `x(2t)`: compressed in time, usually shorter and higher perceived speed.
   - `x(t/2)`: expanded in time, usually longer and lower perceived speed.

### Phase 4: Fourier Transform and Spectrum Analysis

1. Compute the Fourier transform or FFT-based spectrum of `x(t)`, `x(-t)`, `x(2t)`, and `x(t/2)`.
2. Plot magnitude spectra with properly labeled frequency axes.
3. Compare spectra using Fourier transform properties:
   - Time reversal changes phase and mirrors spectral content.
   - Time compression expands the frequency axis.
   - Time expansion compresses the frequency axis.
4. Save all figures under `figures/`.

### Phase 5: Magnitude and Phase Reconstruction

1. Denote `X(jw)` as the Fourier transform of `x(t)`, where `w` is angular frequency.
2. Reconstruct and plot the inverse transform of the magnitude spectrum, `F^-1{|X(jw)|}`.
3. Reconstruct and plot the inverse transform of the phase spectrum, `F^-1{exp(j angle X(jw))}`.
4. Compare both reconstructions with the original signal.
5. Discuss what magnitude and phase each preserve or destroy.

### Phase 6: Frequency-Domain Low-Pass Filtering

1. Choose an ideal low-pass filter cutoff frequency.
2. Apply the filter to `x(t)` in the frequency domain.
3. Compute the inverse transform to obtain the filtered signal.
4. Save the filtered audio clip.
5. Plot and save both waveform and spectrum of the filtered result.
6. Explain how the cutoff frequency affects audio quality and spectral content.

### Phase 7: OFDM/OTFS Paper Study

1. Select one paper from IEEE Transactions on Communications or IEEE Transactions on Wireless Communications about OFDM or OTFS.
2. Save citation details and source notes under `paper/`.
3. Translate the key content.
4. Summarize the paper's technical idea in your own words.
5. Explain why the method can address frequency-selective fading channels or fast mobility scenarios.

### Phase 8: Report Assembly

The report should include:

1. Project objective and selected tools.
2. Audio source and preprocessing details.
3. Interpolation method and waveform of `x(t)`.
4. Time-domain transformations and waveform comparisons.
5. Fourier spectra and theoretical analysis.
6. Magnitude-only and phase-only reconstruction comparison.
7. Low-pass filtering method, cutoff choice, waveform, spectrum, and audio result.
8. OFDM/OTFS paper translation and understanding.
9. Source code explanation and comments.
10. Conclusion.

### Phase 9: Final Packaging and Verification

1. Confirm the report is exported as PDF.
2. Confirm all required audio clips are present.
3. Confirm all source code needed to reproduce results is present.
4. Confirm source code comments meet the 30% notes requirement.
5. Run the code from a clean state to verify figures and audio can be regenerated.
6. Zip the report, audio clips, source code, and necessary supporting files.
7. Submit before June 21, 2026, 23:59.

## 6. MATLAB Implementation Notes

- Main language: MATLAB.
- Recommended entry point: `src/main.m`.
- Suggested helper functions:
  - `src/load_audio.m`: read input audio, convert stereo to mono, trim if needed, normalize amplitude.
  - `src/plot_waveform.m`: save time-domain waveform figures.
  - `src/plot_spectrum.m`: compute FFT and save frequency-domain figures.
  - `src/time_transform.m`: generate `x(-t)`, `x(2t)`, and `x(t/2)`.
  - `src/reconstruct_mag_phase.m`: create magnitude-only and phase-only inverse transform results.
  - `src/ideal_lowpass_filter.m`: apply frequency-domain ideal low-pass filtering.
- Use MATLAB comments generously. The project brief requires at least 30% notes/comments in source code.
